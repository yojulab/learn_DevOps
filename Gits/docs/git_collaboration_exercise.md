# [실습] Git/GitHub 협업 마스터 시나리오

이 실습은 3인 1조(User A, User B, User C)가 되어 GitHub을 통해 하나의 프로젝트를 완성해가는 과정을 시뮬레이션합니다. 각 단계별 미션을 수행하며 협업 흐름을 익혀보세요.

> [!IMPORTANT]
> **준비 사항:** 
> - 조장(User A)은 GitHub에 빈 저장소(Public)를 하나 생성합니다.
> - 조원(User B, User C)은 조장의 저장소 주소를 전달받습니다.
> - 모든 참여자는 자신의 로컬 환경에서 별도의 작업 폴더를 준비합니다.

---

## 단계 1: 프로젝트 시작 및 공유 (User A)
**미션:** 프로젝트 뼈대를 만들고 GitHub에 공유하세요.

1. 로컬 폴더에서 `git init`을 수행합니다.
2. `member.txt` 파일을 만들고 자신의 이름을 적습니다.
3. 첫 번째 커밋을 만들고 GitHub 원격 저장소(`origin`)에 `push` 하세요.

---

## 단계 2: 팀원 합류 (User B, User C)
**미션:** 조장의 프로젝트를 로컬로 가져오고 자신의 정보를 추가하세요.

1. 조장의 저장소를 `git clone` 합니다.
2. `member.txt` 파일을 열어 자신의 이름을 추가합니다.
3. 변경 사항을 커밋하고 `push` 하세요. (누가 먼저 `push` 할지 정해보세요!)
4. 나중에 `push` 하는 사람은 거절(Rejected)될 것입니다. 어떻게 해결해야 할까요?

---

## 단계 3: 기능 병렬 개발 (User A, B, C)
**미션:** 각자 지정된 파일을 생성하여 프로젝트를 확장하세요.

- **User A:** `feature_a.txt` 생성 및 내용 작성
- **User B:** `feature_b.txt` 생성 및 내용 작성
- **User C:** `feature_c.txt` 생성 및 내용 작성

1. 각자 파일을 생성하고 커밋합니다.
2. 원격 저장소의 최신 상태를 유지하면서 각자의 결과물을 `push` 하세요.

---

## 단계 4: 운명의 충돌(Conflict) 발생 (User A, User B)
**미션:** 동일한 파일을 동시에 수정하여 충돌을 유발하고 해결하세요.

1. **User A와 User B**는 `README.md` 파일을 동시에 생성/수정합니다.
2. **User A**가 먼저 내용을 적고 `push` 합니다.
3. **User B**도 다른 내용을 적고 `push`를 시도합니다.
4. **User B**에게 발생하는 오류를 확인하고, 충돌을 해결하여 최종 버전을 `push` 하세요.

---

## 단계 5: 긴급 복구 상황 (User C)
**미션:** 로컬 작업이 완전히 꼬였을 때 원격 저장소 상태로 강제 동기화하세요.

1. **User C**는 `member.txt` 파일을 엉망으로 수정하고 커밋합니다. (Push는 하지 마세요)
2. 작업을 취소하고 싶지만 방법을 모르는 상황을 가정합니다.
3. 원격 저장소(`origin/master`)의 깨끗한 상태를 로컬로 강제 적용(`reset --hard`) 하세요.

---

## 최종 확인 사항
- [ ] 3명의 이름이 모두 `member.txt`에 기록되어 있는가?
- [ ] `feature_a, b, c` 파일이 모두 원격 저장소에 존재하는가?
- [ ] 충돌 해결 후 `README.md`가 정상적으로 보이는가?
- [ ] `git log --graph --all` 명령으로 협업의 흔적을 확인했는가?
